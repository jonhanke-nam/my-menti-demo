> How are we handling saving presentations? And do we log the results as well?

**Claude:**
Read `server/src/routes/sessions.ts`, `server/src/routes/questions.ts`, and `server/src/db/schema.ts` to review the current implementation.

Explained:

**Saving presentations:** Presentations and their questions are saved to the SQLite database (`server/dev.db`) immediately on every action. Creating, editing, deleting ‚Äî all go straight to the DB via the REST API. They persist across server restarts. The data is durable as long as `dev.db` exists.

**Results:** Every vote is saved to the `responses` table in the database (question ID, participant ID, the answer value, timestamp). There's an API endpoint `GET /api/questions/:id/results` that returns aggregated counts for any question. And during a live session, the socket broadcasts full result counts in real-time.

**What's missing:** There's no way for the presenter to review past results after a session ends. The response data is still in the DB, but there's no UI to see it. The `GET /api/questions/:id/results` endpoint exists but nothing in the client calls it outside of a live session.

Identified the gap: **no post-session results review screen**.
